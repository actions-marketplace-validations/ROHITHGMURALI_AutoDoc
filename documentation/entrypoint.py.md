# entrypoint.py

## Overview

This file is the main entry point for the **AutoDoc Swarm GitHub Action**. It orchestrates the entire automated documentation pipeline by:

1. Detecting changed files via Git (smart diffing to save LLM context/costs).
2. Configuring Git user identity so generated documentation can be committed.
3. Creating and invoking an Agent Swarm (Queen, Worker, Drone agents via `create_swarm`) to generate documentation.
4. Committing and pushing any generated documentation back to the repository.
5. Writing execution summaries to the GitHub Step Summary for action-level traceability.

It reads configuration from GitHub Actions environment variables (`GITHUB_*`, `INPUT_*`, `API_KEY`) and uses the `autodoc_swarm.agent.create_swarm` factory to build the multi-agent swarm.

## Functions / Methods

### `get_changed_files(target_dir: str) -> list[str]`

Queries Git to determine which files have changed in the current PR or commit. Used for **smart diffing** to reduce LLM context window usage and costs.

- **Parameters:**
  - `target_dir` (`str`): The target directory to scan. Used to construct the documentation prefix for exclusion.
- **Returns:** `list[str]` — A list of changed file paths, excluding:
  - Files within the `documentation/` subdirectory (to avoid re-documenting already documented files).
  - Common ignore/config files (e.g., `.gitignore`, `.editorconfig`, `.eslintignore`).
- **Behavior:**
  - On a **pull_request** event: fetches the base branch (`GITHUB_BASE_REF`) and runs `git diff --name-only origin/{base_ref}`.
  - On other events (e.g., push): runs `git diff --name-only HEAD~1 HEAD` to get changes from the latest commit.
  - If `subprocess.CalledProcessError` is raised (e.g., first commit with no history), prints a warning and returns an empty list, triggering a full scan downstream.

---

### `configure_git() -> None`

Configures global Git identity and safe directory settings so the action can commit and push changes.

- **Parameters:** None (reads `GITHUB_WORKSPACE` from environment variables).
- **Returns:** `None`
- **Behavior:**
  - Sets `user.name` to `"AutoDoc Swarm"`.
  - Sets `user.email` to `"autodoc-swarm@github.actions"`.
  - Adds `GITHUB_WORKSPACE` to Git's `safe.directory` whitelist if `GITHUB_WORKSPACE` is set.
  - All `git config` commands are run with `check=True`, raising `CalledProcessError` on failure.

---

### `commit_and_push(target_dir: str) -> bool`

Commits generated documentation to Git and pushes it to the remote branch.

- **Parameters:**
  - `target_dir` (`str`): The root directory. Documentation is staged from `{target_dir}/documentation/`.
- **Returns:** `bool` — `True` if changes were committed and pushed successfully; `False` if there were no changes to commit.
- **Behavior:**
  1. Ensures `{target_dir}/documentation/` exists (creates it if missing).
  2. Stages the documentation directory with `git add`.
  3. Runs `git status --porcelain` to check for staged changes — returns `False` early if none exist.
  4. Commits with message `"docs: auto-generated by Swarm Queen"`.
  5. Determines the target branch from environment variables:
     - `GITHUB_HEAD_REF` for pull requests.
     - `GITHUB_REF_NAME` for pushes.
  6. Pushes to the remote using a `https://x-access-token:{token}@github.com/{repo}.git` URL, requiring `GITHUB_TOKEN`, `GITHUB_REPOSITORY`, and `branch` to all be set.
  7. Exits the process (`sys.exit(1)`) if any Git operation fails or required environment variables are missing.

---

### `write_step_summary(content: str) -> None`

Appends content to the GitHub Action's step summary file for traceability in the GitHub Actions UI.

- **Parameters:**
  - `content` (`str`): The markdown-formatted string to append.
- **Returns:** `None`
- **Behavior:**
  - Reads the `GITHUB_STEP_SUMMARY` environment variable for the summary file path.
  - If set, opens the file in append mode and writes the content followed by a newline.
  - Silently does nothing if `GITHUB_STEP_SUMMARY` is not set.

---

### `main() -> None`

The orchestrator function that drives the entire AutoDoc Swarm pipeline. Called when the script is executed as `__main__`.

- **Parameters:** None (reads all configuration from environment variables).
- **Returns:** `None` (exits via `sys.exit(1)` on error).
- **Environment Variables Read:**
  | Variable | Default | Description |
  |---|---|---|
  | `INPUT_PROVIDER` | `"openrouter"` | LLM provider to use. |
  | `INPUT_QUEEN_MODEL` | `"anthropic/claude-3.5-sonnet"` | Model for the Queen agent. |
  | `INPUT_WORKER_MODEL` | `"anthropic/claude-3.5-sonnet"` | Model for the Worker agent. |
  | `INPUT_DRONE_MODEL` | `"anthropic/claude-3.5-sonnet"` | Model for the Drone agent. |
  | `INPUT_TARGET_DIR` | `"."` | Target directory to scan and document. |
  | `API_KEY` | *(required)* | API key for the LLM provider. |
  | `GITHUB_WORKSPACE` | — | GitHub Actions workspace directory. |
  | `GITHUB_EVENT_NAME` | — | Event type (`pull_request` or push). |
  | `GITHUB_BASE_REF` | — | Base branch ref (for PRs). |
  | `GITHUB_HEAD_REF` | — | Head branch ref (for PRs). |
  | `GITHUB_REF_NAME` | — | Ref name (for pushes). |
  | `GITHUB_TOKEN` | — | Auth token for pushing. |
  | `GITHUB_REPOSITORY` | — | Repository slug (`owner/repo`). |
  | `GITHUB_STEP_SUMMARY` | — | Path for step summary output. |

- **Execution Flow:**
  1. Validates that `API_KEY` is provided; exits if missing.
  2. Exports the provider-specific API key environment variable (e.g., `OPENROUTER_API_KEY`).
  3. Changes the working directory to `GITHUB_WORKSPACE`.
  4. Calls `configure_git()` to set up Git identity.
  5. Calls `get_changed_files(target_dir)` for smart diffing.
  6. Constructs a kickoff message — either targeting specific changed files or requesting a full repository scan.
  7. Calls `create_swarm()` to instantiate the agent swarm.
  8. Invokes the Queen agent with the kickoff message.
  9. Extracts the final output from the Queen's response.
  10. Writes the final output to the GitHub Step Summary.
  11. Calls `commit_and_push(target_dir)` to persist documentation changes.
  12. Writes a success/no-changes summary based on the push result.
  13. On any exception, writes an error summary and exits with code 1.
